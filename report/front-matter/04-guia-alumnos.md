# Curso de Fundamentos de Java para Principiantes {-}

## Resumen del Curso {-}

Este curso de 1 hora introduce a estudiantes de secundaria a los fundamentos de la programación con Java, incluyendo programación orientada a objetos. *No requiere descargas*, solo abrir el navegador web.

**Duración total**: ~60 minutos  
**Público Objetivo**: Estudiantes de 12 a 17 años sin experiencia en programación  
**Prerrequisitos**: Ninguno  
**Herramientas necesarias**: **Navegador web** (Chrome, Firefox, Safari, Edge, Brave)  
**Repositorio de Código Fuente**: [https://github.com/upc-pre-1asi0729-11848-arcadiadevs/java-fundamentals-course-arcadiadevs](https://github.com/upc-pre-1asi0729-11848-arcadiadevs/java-fundamentals-course-arcadiadevs)

## Ruta de Aprendizaje Recomendada {-}

Para aprovechar al máximo este curso, te recomendamos seguir los siguientes pasos en cada lección:

1. **Mira el video explicativo:** Accede al video de YouTube correspondiente a la lección. Presta atención a los conceptos teóricos y a cómo el instructor escribe el código.
2. **Abre el código inicial:** Cada clase tiene un enlace directo a **OnlineGDB** con el código inicial ya precargado. Ábrelo en otra pestaña de tu navegador.
3. **Escribe y experimenta:** Acompaña la explicación escribiendo el código tú mismo. ¡No te limites a copiar y pegar! La memoria muscular es clave en programación.
4. **Resuelve el Reto Práctico:** Al final de las notas de cada lección encontrarás un pequeño ejercicio diseñado para evaluar lo aprendido. Intenta resolverlo sin ver el código de solución.

## Estructura del Repositorio {-}

El repositorio oficial del curso en GitHub está organizado para facilitarte el acceso a los recursos y archivos de trabajo de cada una de las clases:

- **`starter-files/`**: Contiene la plantilla inicial en Java (Main.java) para cada lección. Es el punto de partida en tu editor en línea antes de empezar a programar.
- **`completed-examples/`**: Contiene el código completamente terminado y documentado con comentarios detallados al finalizar cada clase. Utilízalo como guía de referencia si tienes algún error de compilación.
- **`report/`**: Contiene todo el material de estudio impreso o en formato digital, incluyendo las notas completas de cada clase con ilustraciones explicativas.

## Consejos para Aprender a Programar {-}

- **No temas a los mensajes de error:** Los errores no significan que seas malo programando; son las herramientas que usa la computadora para decirte exactamente qué parte de las "instrucciones" no comprende. Léelos con atención: a menudo indican la línea exacta y el símbolo faltante (como un `;` o un `}`).
- **Comenta tu código:** Usa los comentarios (`//`) para escribir notas en tu propio lenguaje explicando qué hace cada línea. Esto te ayudará a repasar tu lógica en el futuro.
- **Modifica y rompe las cosas:** Una vez que el programa funcione, intenta cambiar números, textos o variables para ver cómo reacciona. Es la mejor forma de entender el comportamiento real del código.
